### 3f.1 Device Power State Before Register Access

MMIO/system-register access is valid only while required power domains, clocks,
resets, and performance votes are in the required state.

| Trigger | Required check / proof | Finding |
|---|---|---|
| pm_runtime bracket: register access reachable outside enable/disable paths, such as sysfs, debugfs, or IRQ | Hold a PM runtime reference or prove the power domain is guaranteed on. Prefer `pm_runtime_resume_and_get(dev)`: check negative return, access no registers on failure, and release with `pm_runtime_put(dev)` on success. | Unchecked PM resume before register access is `[BUG]`. |
| `pm_runtime_get_sync(dev)` | It increments usage before resume. On negative failure, balance with `pm_runtime_put_noidle(dev)`, not `pm_runtime_put()` or `pm_runtime_put_sync()`. Treat only negative returns as errors; `1` can mean successful wake. After a successful get, every newly added `return` or `goto` before the common `pm_runtime_put*()`/disable cleanup must be routed through cleanup or proved to execute before the get succeeds. | Missing return check, missing `put_noidle`, `if (ret)` treating success as failure, or a post-get early return that leaks the usage count is `[BUG]`. If diff/context contains `pm_runtime_get_sync(`, the review must explicitly state checked return, error balancing, post-success cleanup, or preferred migration to `pm_runtime_resume_and_get`; silence is a validator violation. |
| Probe-time capability reads | Read hardware ID/capability registers only after power/clock enable and before configuration. Cache only static capabilities; dynamic capabilities such as negotiated speed or firmware-reported mutable features must be re-read on each probe. | Runtime state cached as capability data is `[CONCERN]`. |
| Power-state restore symmetry: suspend/runtime_suspend/deactivate power-collapses hardware | Resume/activate must restore every non-retained state: registers, clocks, resets, ICC/interconnect bandwidth, regulators, OPP/genpd/perf votes, DMA state, and cached software mirrors. Name the exact resume/helper call that restores each dropped vote/state. | Reachable deterministic missing restore is `[BUG]`; use `[CONCERN]` only when retention or reachability remains uncertain after reading the resume helper and transfer path. |
| cached fast path: restore deferred to next transfer | Quote any cached fast path such as `if (clk_hz == cur_speed_hz) return 0;` and prove which call defeats it. Do not claim a helper restores OPP/genpd votes if its comment/body says it does not alter performance states. | If the cached fast path can skip restore/revote/reprogram, file `[BUG]`/`[CONCERN]`; silence on the cached fast path is a validator violation. |
| Global vote scope vs per-block helpers: global OPP/perf/ICC/genpd vote raised/dropped inside per-block or per-core helper | Trace the full multi-block sequence. To clear `per_block_vote_scope`, name every sibling block and prove none remains active when the global vote drops. Local symmetry, “runtime semantics unchanged”, or “symmetric with enable” is not proof. | Vote reduction while a sibling can remain active is `[BUG]`. |
| Failed-start / failed-resume stale-state audit: updates cached rate, enabled flag, ownership, or accounting before setup completes | Inspect the failure exit and next transfer/fast path. | Later shortcuts trusting stale software state and skipping real restore/start are `[BUG]`. |
| Touched-file PM source-aware audit: any touched file contains a bare `pm_runtime_get_sync(` (in the diff hunks, in surrounding context, or anywhere in the unchanged source the patch touches), regardless of whether the patch added the call | **Decisive evidence (all three required for any finding or dismissal):** (1) the touched-file path and the line where `pm_runtime_get_sync(` appears (quote `path/file.c:NNN`); (2) the function reachability of that line — name the entry point (sysfs/debugfs/ioctl/IRQ/probe/runtime callback) that can drive execution to it; (3) the return-handling and balance evidence read from the same source file. **Valid dismissal proofs (cite source for each):** the return value is checked and `< 0` flows to `pm_runtime_put_noidle(dev)` (quote both lines, same path); the call has been migrated to `pm_runtime_resume_and_get(dev)` already (quote the migrated line — the audit no longer applies); the bare site is on a path the touched diff cannot reach (quote the entry-point control flow that proves unreachability — paper-link claims like "only called at probe with PM disabled" are not enough; show the gate); the device's runtime-PM is unconditionally suppressed via `pm_runtime_disable()` for the lifetime of the access (quote the disable site and the access site under the same lock/state). **Disqualified dismissals — file the finding instead if any of these is the only argument:** "the put_sync on the error path balances it" (a failed `pm_runtime_get_sync` increments usage and `pm_runtime_put_sync()`/`pm_runtime_put()` does not undo a non-resume; only `pm_runtime_put_noidle` does — quote `put_noidle` or migrate); "the return is checked" without quoting the comparison line (`if (ret < 0)` etc.) and the path the failure case takes; "pre-existing, not introduced by this patch" — the audit fires on touched files regardless of who wrote the line, because the patched execution path now reaches it; "ret == 1 means already-resumed so the check is correct" without quoting the actual check (the rule is `ret < 0`, treating `1` as success); "matches the rest of the driver" without quoting one balanced peer site; "the framework prevents bad state" without quoting the framework call that does it. | Bare `pm_runtime_get_sync(` reachable on a touched path without source-cited return check + put_noidle balance is `[BUG]`. For any dismissal, the report must quote either the migration to `pm_runtime_resume_and_get`, the `if (ret < 0)` + `put_noidle` pair from the same file, or the unreachability gate — without one of those quotes, file the finding. |
