# Reviewer Packet Schema

Reviewer packets are generated by the orchestrator for one patch. They are the
sub-agent data-plane input and must stay free of control-plane workflow text.

## Markdown Packet Sections

Required sections:

1. `packet-metadata`
   - schema version;
   - mode;
   - slug;
   - patch index and total;
   - commit hash and subject;
   - changed files;
   - selected rule-card IDs.
2. `reviewer-base`
   - contents of `refs/reviewer-base.md`.
3. `output-format-mini`
   - contents of `refs/output-format-mini.md`.
4. `context-snippets`
   - bounded source snippets around changed lines plus rule-card-driven context probes chosen by the orchestrator.
5. `context-coverage`
   - per-card inventory of required context keys, whether candidate evidence is in the packet, and what remains missing.
6. `selected-rule-cards`
   - zero or more triggered cards from `refs/rule-cards/`.
7. `commit-message`
   - full commit message when available.
8. `patch-diff`
   - patch diff or a clear missing-diff marker.
9. `checker-evidence`
   - paths to build/check/test artifacts, not long workflow instructions.

## JSON Sidecar Fields

The optional JSON sidecar mirrors machine-readable packet facts:

```json
{
  "schema": "review-commits.reviewer-packet.v1",
  "mode": "A",
  "slug": "example",
  "patch": 1,
  "total": 3,
  "hash": "...",
  "subject": "...",
  "files": ["path"],
  "paths": {
    "diff": "tmp/patch_1_diff.txt",
    "packet": "tmp/patch_1_review_packet.md",
    "packet_json": "tmp/patch_1_review_packet.json"
  },
  "rule_cards": [
    {
      "id": "qcom-scm-vmid-memory-assignment",
      "card": "refs/rule-cards/qcom-scm-vmid-memory-assignment.md",
      "triggers": ["qcom_scm_assign_mem"],
      "requires_context": ["allocation_site", "assign_unassign_call_sites"]
    }
  ],
  "context_snippets": [
    {
      "file": "drivers/example.c",
      "start": 10,
      "end": 50,
      "reason": "context coverage: qcom-scm-vmid-memory-assignment/allocation_site"
    }
  ],
  "context_coverage": [
    {
      "card": "qcom-scm-vmid-memory-assignment",
      "required": [
        {
          "name": "allocation_site",
          "status": "evidence_in_packet",
          "evidence": ["drivers/example.c:10-50"],
          "search_terms": ["qcom_scm_assign_mem"],
          "notes": []
        },
        {
          "name": "free_or_unprepare_paths",
          "status": "missing_from_packet",
          "evidence": [],
          "search_terms": [],
          "notes": []
        }
      ],
      "missing": ["free_or_unprepare_paths"]
    }
  ]
}
```

## Context Coverage Semantics

`context_coverage` is a packet evidence inventory, not a safety proof.

- `evidence_in_packet`: the orchestrator found candidate lines/snippets for the
  required context key and included them in the packet. The reviewer must still
  reason over the evidence.
- `missing_from_packet`: the orchestrator did not include candidate evidence for
  that key. The reviewer must mark the specific card check inconclusive if the
  missing fact controls the decision.
- Missing context is not a packet-size or review-failure gate by itself; it is
  review-quality telemetry and can drive a future orchestrator-mediated retry.

## Forbidden Content

Packets must not include:

- `refs/startup-workflow.md`;
- `refs/orchestrator-workflow.md`;
- `refs/html-template.md`;
- validator implementation details;
- sub-agent spawning/retry instructions;
- b4/git/apply workflow;
- Mode C workflow during Mode A/B review;
- broad legacy rule briefs such as `core.md`, `gate-rules.md`, `code-logic.md`,
  `hardware-eng.md`, or their `code-logic-*` / `hardware-eng-*` fragment files
  unless they have been converted into selected cards. The generic bug-class
  rules those briefs once carried (ERR_PTR guards, atomic-context sleeps, DMA
  ownership, stored stack addresses, RCU/refcount release, resource
  acquire/release symmetry, programming sequences, per-CPU/hotplug safety,
  register-field access, uninitialized structs, unchecked setup results, lock
  symmetry, probe-defer/export/topology) are now `refs/rule-cards/*` selected by
  `select_rule_cards.py` for Mode A/B packets; the legacy `refs/code-logic-*.md`
  briefs remain available as background reference but are not loaded into
  Mode A/B packets.
