### 3f.2 Hardware Resource Lifecycle Symmetry

Every acquired hardware resource must be released on all error, disable, remove,
offline, and recovery paths unless ownership is deliberately transferred to a
named later cleanup path.

| Resource / pattern | Required check / proof | Finding |
|---|---|---|
| IRQ lines | `request_irq()` requires `free_irq()` on all error/remove paths. `devm_request_irq()` must not be manually freed in `.remove()`. | Manual `free_irq()` for a devm IRQ is `[BUG]`. |
| DMA channels | `dma_request_chan()` requires `dma_release_channel()` on all error/remove paths. | Missing release is `[BUG]`. |
| Clocks/regulators | Pair `clk_prepare_enable()` with `clk_disable_unprepare()` and `regulator_enable()` with `regulator_disable()`. Any non-devm reference getter (`of_clk_get()`/`of_clk_get_by_name()`/`clk_get()`, `regulator_get*()`, `of_reset_control_get*()`, etc.) returns an owned handle: pair it with the matching `*_put()` on every error, teardown, suspend-disable, and unbind path, or convert to the `devm_*` variant when device lifetime matches. `devm_clk_get_enabled()` and similar auto-disable helpers must not be manually disabled. | Missing pair, missing release for an owned reference handle, or double-disable is `[BUG]` unless a named later owner/lifetime transfer is proved. |
| Repeatable enable paths: any callback that the framework can invoke more than once per device session — open/close, hw_params/start/stop, set-clock/set-rate, runtime-PM resume/suspend, recovery, switch reconfigure, stream restart | Audit enable-count idempotency for every `*_prepare_enable()` / `*_enable()` / `*_get_enabled()` introduced or reordered in the path. A path that can re-enter must have a real per-user refcount, an `enabled`/`prepared` guard that prevents duplicate prepares, or a framework one-call guarantee documented for that callback; the matching shutdown/error path must disable exactly the count acquired. | Repeated prepare/enable with only one disable is `[BUG]` when the callback is reachable through repeated or concurrent sessions; `[CONCERN]` when reachability depends on framework call ordering. |
| Firmware/bootloader already-on state with software-managed resources | If a framework status callback can report hardware already enabled before Linux has called the driver enable path, synchronize Linux-side resource counts at probe/registration or make disable paths conditional on a tracked software enable. Hardware-on status does not prove `clk_prepare_enable()`, `pm_runtime_get*()`, reset deassert, or bus-vote counts are held. | A later `*_disable_unprepare()` / PM put / reset assert without a matching Linux-side acquire is `[BUG]` when reachable; `[CONCERN]` when firmware state is plausible but not proven. |
| Wrappers that translate framework helpers returning `≥0`-on-success | Some framework calls (notably `pm_runtime_put_sync()`, `pm_runtime_put()`, `pm_runtime_resume_and_get()`'s sibling helpers, certain `regmap` operations) can legitimately return positive values on success. Wrappers whose contract is `0`/negative-errno must clamp positive returns to 0 (`return ret < 0 ? ret : 0;` or `max(ret, 0)`) unless every caller documents and handles positive values. | Forwarding a positive success value from an errno-style wrapper is `[MINOR]`/`[CONCERN]` depending on caller behavior. |
| `ioremap()` | Pair with `iounmap()` on error/remove paths. | Missing unmap is a resource leak. |
| `devm_*` in repeatable paths | Probe-time one-shot devres is safe. In open/close, firmware reload, recovery, stream restart, or retry paths, identify the owning device and whether it persists across retries. | Repeated allocation on a persistent device without teardown/reuse is `[BUG]`. |
| `device_unregister()` / `put_device()` of caller-owned object | Clear every caller-owned pointer/flag that later init/error/teardown can test. `device_unregister()` does not NULL the caller pointer. To clear `device_unregister_pointer_hygiene`, quote the exact `<ptr> = NULL` or equivalent reset after unregister on the same path. | If no reset line exists and stale pointer can be reused, file `[BUG]` for stale pointer/UAF-on-reinit. Positive prose, “cleanup is symmetric”, or “mutually exclusive branches” is not proof. |
| Transport-backed regmap bounds: `regmap_raw_read` / `reg_read` | Reject sizes larger than the protocol response buffer before queuing async transport. | Copying a fixed response and returning success leaves stale caller bytes. |
| Per-instance hardware resources such as trace IDs, DMA descriptors, slots | Release on disable, unbind, CPU/hardware offline, and enable-error paths. | Missing release corrupts shared hardware state and is `[BUG]`. |
| Active session teardown before device unregistration | `.remove()`/unbind must tear down trace/DMA/ring/session state and clear per-CPU references before `device_unregister()` or equivalent. `devm_*` does not manage software session state. | Leaving per-CPU/device refs to freed objects is `[BUG]`. |
| Cross-instance raw pointer held across independent unbind: one driver instance/provider stores a raw `struct *` into a sibling instance's allocation (peer node, cross-provider link, shared topology, foreign device object) | **Decisive evidence (all three required for any finding or dismissal):** (1) the sibling-pointer storage site in the diff (e.g. `icc_link_nodes(src, &sibling->node)`, copying a `struct device *`, an inbound `void *priv` retained on a sibling); (2) the unbind path of the *target* instance (the framework destroy/release that frees the object behind that pointer — read it from source, do not infer); (3) whether the source instance has a back-reference clear or lifetime guarantee that runs *before* the target frees. **Valid dismissal proofs (cite source for each):** `.suppress_bind_attrs = true` on both driver structs (quote both); refcount/`get_device()` on the target with paired `put_device()` on source teardown (quote both); a managed `device_link` between the two devices that the framework auto-purges (quote the `device_link_add` and the consumer/supplier flags); framework-side back-reference cleanup that NULLs inbound pointers on destroy (quote the framework function and the line that does the clear). **Disqualified dismissals — file the finding instead if any of these is the only argument:** "devm cleans up" without quoting which devm action invalidates the *sibling's* retained pointer (devm runs on the device that owns the pointer, not the target); "removed together" without quoting an enforcement that prevents independent sysfs unbind; "mutually exclusive" without quoting the locking/ordering invariant; "framework handles it" without quoting the framework function and showing it touches sibling state; same-driver / same-module assumptions (a module-level relation does not bind two `platform_device`s together — sysfs unbind is per-device); "the core would invalidate the pointer" without reading the core's destroy path and quoting the line that does the invalidation. | A peer raw pointer that survives the target's independent unbind and is later dereferenced (path-find, traversal, callback) is `[BUG]` (UAF). For any dismissal, the report must quote the lifetime guarantee, the back-reference clear, or the bind-suppression line — without one of those quotes, file the finding. |
| Read-path widening of a writer-locked pointer (trigger: a patch adds new dereferences of a struct field inside an existing `if (field)` test or other unlocked read site, extending the field's use beyond a single check). The pre-patch code typically used the field once (a single deref or pass to one helper). The patch turns that into a multi-step block — pass to one helper, then call another, etc. — so the field is *live* across more code than before. | **Bad-pattern shape (subsystem-agnostic):** `if (ctx->ptr) {` ➜ `if (ctx->ptr) { use_a(ctx->ptr); use_b(arg); return ...; }` while a sibling teardown function (`*_unplugged()`, `*_disconnect()`, `*_release()`, `.remove`, an HPD/work/threaded-IRQ/timer handler) calls `kfree(ctx->ptr)` / `*_free(ctx->ptr)` / `devm_kfree`/`put_device` on the same field, and the writer is reachable concurrently with the reader. **Decisive evidence (all three required):** (1) the widened read block in the diff — quote the new dereference(s) the patch added inside the existing nullness test, naming the field; (2) the writer site in source — quote the line that frees / nulls the same field (`kfree(ctx->buf)`, `*_free(ctx->ptr)`, `put_device(ctx->dev)`) and the function it lives in; (3) the lock the writer holds when freeing — quote the `guard(mutex)(&X)` / `mutex_lock(&X)` / `spin_lock_irqsave` *and* show that the patched read function does NOT acquire the same lock (no `guard(mutex)`, no `mutex_lock`, no RCU read-side, no refcount on the object). **Valid dismissal proofs (cite source for each):** the read function is itself called only with the same lock held by every caller (quote every call site and its lock acquisition); the field is RCU-protected with the writer using `synchronize_rcu()` / `kfree_rcu()` and the reader inside `rcu_read_lock()` (quote both); the field carries its own refcount (`kref_get`, `get_device`, or any object-specific `*_dup`/`*_get` that increments) acquired before the deref window and released after (quote the get/put pair); the writer is single-threaded with the reader by hardware/topology guarantee (quote the producer's serialisation — e.g. a single workqueue, suppress_bind_attrs that prevents disconnect, or an ordering mutex held across both); the patched block returns / completes synchronously *under* a lock the writer also takes that the original code obviously held (quote the caller's locking that brackets the read). **Disqualified dismissals:** "the writer is rare / hotplug is uncommon" — frequency does not eliminate UAF; "the original `if` test was already unlocked" without quoting that the *original* code only read the pointer once and the widening adds new derefs that lengthen the live window; "framework core handles it" without quoting the core function and showing it serialises this exact field; "the new path matches the existing path" / "the fix mirrors the existing pattern" — pattern alignment is a correctness-of-API argument, NOT a lifetime argument; the writer can still free the pointer regardless; "fix is correct and complete" / "API contract satisfied" without naming the writer's free site at all; "no new dereferences in the diff" when the diff demonstrably adds at least one new use (quote the added line). **Severity:** `[BUG]` when the writer is reachable from a separate execution context (HPD/work/threaded IRQ/timer/sysfs) and frees the same field — concurrent-UAF risk newly widened by the patch; `[CONCERN]` when the writer is reachable but the deref window is very short (single helper call) and downgraded by Gate 3 race-likelihood analysis. **Required record format:** under `codebase audit: callees` for the widened read, name the writer function, the line that frees the field, and the lock it holds; under `codebase audit: siblings`, state whether the read function holds the same lock or cite the lifetime guarantee. | A read-path widening that extends derefs of a writer-freed pointer without holding the writer's lock — and without an alternate lifetime guarantee — is `[BUG]`. The `if (ptr)` test cannot prove the pointer is still valid by the time the *next* line uses it. |
| Queued work before resource teardown: work or async completion can touch resources | `.remove()` and error unwind must `cancel_work_sync()` / `cancel_delayed_work_sync()` before regmap exit, buffer free, IRQ free, or transport unregister. Also quiesce timers, hrtimers, tasklets, completions, and kthreads before freeing resources they can touch. | `devm_*` cleanup does not cancel ordinary async execution. |
| regmap_init / regmap_exit pairing: bare `regmap_init()` | Prefer `devm_regmap_init*()`. Before `regmap_exit()`, cancel every work item touching it, unregister every callback/client that can invoke it, and reject/signal in-flight completion paths. Same for probe errors. | `regmap_exit()` while live callback/work can touch it is `[BUG]`. |
| Idempotency governs severity assessment, not code design | Idempotency may reduce Gate 2 harm when assessing an asymmetric error path, but it is not a caller contract and must not justify designing away a release. | Refactor-introduced release asymmetry is at least `[MINOR]`; idempotency can rule out `[BUG]`/`[CONCERN]` but not reduce below `[MINOR]`. |
| Disable path must be read before filing a resource-leak finding | Read the corresponding disable/teardown path before filing. Scenario A — consistent omission/design intent: if disable and all error paths consistently omit release by documented design, dismiss entirely and document the invariant. Scenario B — refactor-introduced asymmetry: if a refactor moved release so one error path no longer releases while another does, apply the `[MINOR]` behavioral-regression floor. | Do not file even `[NIT]` for confirmed intentional consistent omission; do not drop refactor asymmetry below `[MINOR]`. |
| Unpublished-resource unwind / fallback audit | Name acquisition site, publication site, and every pre-publication return. For fallback parsers, state whether the next parser reuses the handle or requires fresh acquisition. | Dismiss only when each path releases exactly once or transfers ownership to a named later cleanup path. |
| Duplicate teardown / unregister / release audit | Enumerate every shutdown, release, unregister, disable, and free operation and confirm each affected resource instance sees it exactly once per reachable path. | Missing or duplicate teardown is a resource-lifecycle bug. |
| Required resource absence | If match data or binding text says a compatible requires clocks, power domains, regulators, resets, or bus enables, probe must fail clearly when none are acquired. | Letting invalid DT continue to later MMIO/PHY/register access is a hardware bring-up bug or concern even when schema should catch the DT mistake. |
